# 問い合わせ導線の整備 — 設計書

最終更新: 2026-05-01
ステータス: ドラフト（実装着手前）

## 背景

`goodwithsite` の CONTACT セクションは現状、対応形態（顧問 / スポット / プロジェクト）の明示と、メール（`info@goodwith.tech`）+ X DM の連絡先リスト構成。

`docs/plans/2026-04-30-site-relaunch.md` のフェーズ3「問い合わせフォームの実装」を本仕様で扱う。

ただし、本人がフォーム送信に魅力を感じていないこと、ターゲット顧客（事業会社情シス・SaaS・大企業クラウド移行PJ・官公庁/規制業種）にメール文化が強いこと、戦略上「最初の接点を増やす」のが正である（asset-plan の「インバウンド一本」戦略）ことを踏まえ、**フォームを置かず、気軽に連絡できるチャネルを増やす方針**に転換する。

## ゴール

- 訪問者が **「15分相談予約 / メール / X DM」のいずれかで気軽に連絡できる** 状態を作る
- 顧客の状況（メール文化の堅さ、稟議の要否、軽い壁打ちか）に応じて自然に選ばれる導線を提供する
- 既存の戦略文言（「コンテナセキュリティを設計から実装まで」「守秘契約前提」）と整合する
- サイトに外部スクリプトを読み込まず、セキュリティ業としての整合性を保つ

## 非ゴール

- 自前バックエンドでのフォーム処理（Vercel Serverless Function、Resend 等）の実装
- 第三者フォームサービス（Formspree 等）の導入
- iframe 埋め込みによる予約UIのインライン表示（外部リンク方式に統一）
- 多言語対応（既存サイトと同様、日本語ベース）
- スパム対策の専用実装（mailto と外部リンクのみのため不要）
- 予約後の自動返信メール文面のカスタマイズ（TimeRex デフォルトを使用）

## 採用する導線の構成

### 3チャネル並列カード

CONTACT セクション内に、以下3つを等価に並べる:

| チャネル | 目的 | リンク先 |
|---|---|---|
| 15分相談を予約 | 軽い壁打ち、設計レビュー、技術選定の議論（必要に応じて延長） | TimeRex 予約ページ（外部リンク、新規タブ） |
| メールで連絡 | 公式記録、社内稟議が要る顧客、見積もり相談 | `mailto:` 件名+本文プリフィル |
| X DM で連絡 | カジュアル質問、OSS 文脈、知り合い経由 | `https://x.com/tomoyamachi`（既存維持） |

### 全体構成（CONTACT セクション）

```
[Title: CONTACT]

[Lead]
コンテナセキュリティ・クラウドセキュリティに関するご相談を承ります。
守秘契約前提でお話を伺いますので、検討段階のご相談もお気軽にどうぞ。

[対応形態 3カード]（既存維持）
顧問契約 / スポット相談 / プロジェクト参画

[連絡先 3チャネルカード]（新規）
15分相談を予約 / メールで連絡 / X DM で連絡
```

### 3チャネルカードの内容

各カードは、見出し（チャネル名）、補足文（用途）、ボタン（アクション）の3要素で構成する。

**カード1: 15分相談を予約**

- 見出し: `15分相談を予約`
- 補足: `オンライン15分から、Google Meet 等で対応します（指定可・延長可）`
- ボタン: `日程を選んで予約 →`
- リンク: TimeRex 予約ページ URL（プレースホルダー）、`target="_blank" rel="noopener noreferrer"`

**カード2: メールで連絡**

- 見出し: `メールで連絡`
- 補足: `件名・本文テンプレート付き。社内共有・記録に向きます`
- ボタン: `メールを書く →`
- リンク: `mailto:info@goodwith.tech?subject=...&body=...`

**カード3: X DM で連絡**

- 見出し: `X DM で連絡`
- 補足: `カジュアルなご質問・OSS 文脈のお声がけに`
- ボタン: `@tomoyamachi へ送る →`
- リンク: `https://x.com/tomoyamachi`、`target="_blank" rel="noopener noreferrer"`

## mailto テンプレート仕様

### 件名

```
[GOODWITH] お問い合わせ
```

`[GOODWITH]` プレフィックスにより、本人の受信箱で識別しやすくする。

### 本文

```
天地様

(自己紹介・社名)

(ご相談内容)

(対応形態のご希望: 顧問契約 / スポット相談 / プロジェクト参画 / 未定)

----
※ 守秘契約前提でお話を伺います。検討段階のご相談もお気軽にどうぞ。
```

URL エンコード時の改行は `%0A`、空白は `%20` を使用する。実装では `encodeURIComponent` を用いる。

## TimeRex 予約ページの設定方針

### サービス・プラン

- TimeRex 無料プラン
- 月10件規模を想定（無料プランの月100調整制限内に十分収まる）

### 予約ページの設定（本人が TimeRex 上で実施）

- カスタムスラッグ: `goodwith` または `tomoyamachi`（無料プランで設定可）
- イベント: 15分相談（必要に応じ延長）、Google Calendar 連携、Google Meet 自動生成
- 予約フォーム カスタム項目:
  - 会社名（必須）
  - 役職（任意）
  - 相談したい内容（自由記述、必須）
  - 想定する関わり方（顧問 / スポット / プロジェクト / 未定、任意）

サイト実装時点では TimeRex 予約ページ URL はプレースホルダ `https://timerex.net/s/PLACEHOLDER` で記述し、実装PRマージ前に本人が TimeRex で発行した実URLに差し替える。

### URL の管理

サイト側コードにハードコードする。環境変数化はしない（公開URLであり秘匿性ゼロのため）。

## レイアウト・スタイル方針

既存の対応形態 3カード（`engagementGrid` クラス）と視覚的にリズムを揃える。

### グリッド

- デスクトップ: 3カラム（既存 `engagementGrid` と同じ `grid-template-columns: repeat(3, 1fr)`）
- モバイル: 1カラム（既存と同じ `@media (max-width: 600px)` で `grid-template-columns: 1fr`）

### カード内のスタイル

- 背景: `#fff`
- パディング: 既存 `engagement` クラスと同じ `22px 20px`
- 角丸: `8px`
- 影: 既存と同じ `0 1px 3px rgba(0, 0, 0, 0.06)`
- 見出し色: `#2096a3`（サイト基調色、既存維持）
- ボタン:
  - 文字色: `#2096a3`
  - 背景: 透明
  - 枠線: `1px solid #2096a3`
  - パディング: `8px 16px`
  - hover: 背景 `#2096a3`、文字色 `#fff`
  - ボタンスタイルは新規追加、既存に同等のものがないため

### 既存リスト（`channels`, `channelLabel`, `channels li`）の扱い

3チャネルカード化に伴い、既存の `.channels`, `.channelLabel`, `.channels li` セレクタは未使用となる。CSS から削除する。

## アクセシビリティ・SEO

- 各ボタンは `<a>` 要素として実装し、外部リンクには `rel="noopener noreferrer"` と `target="_blank"` を付与
- `mailto:` リンクは新規タブを開かない（メーラー起動が期待動作のため）
- カードの見出しは `<h4>` で既存対応形態カードと同じ階層
- 各カードの補足文は `<p>`、ボタンは `<a>`
- `aria-label` は不要（リンクテキストで十分意図が伝わる）

## 戦略との整合性

| 戦略要素 | 本設計での反映 |
|---|---|
| 「インバウンド一本」「最初の接点を増やす」 | 3チャネル並列で接点増加、フォームによる離脱を発生させない |
| 「コンテナセキュリティを設計から実装まで」 | 15分の軽い壁打ちから始め、必要に応じて延長して設計レビューまでをカバー |
| 守秘契約前提 | リード文に既存の文言維持、メールテンプレ末尾にも明記 |
| ターゲット顧客の多様性（情シス・SaaS・官公庁） | 各層の文化に合うチャネルを並列提供 |
| セキュリティ業としての整合性 | 自サイトに外部スクリプトを読み込まない（TimeRex はリンク方式） |
